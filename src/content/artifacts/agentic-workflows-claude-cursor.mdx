For a decade, the industry didn't know what to do with us.
Designers told us we "cared too much about code."
Engineers told us we "cared too much about pixels."

**In 2026, we finally get the respect we deserve.**

This is our "Think Different" moment. The AI revolution didn't replace engineers; it automated the *boring* parts of engineering, effectively clearing the path for the **Senior Design Engineer**. We are the only ones equipped to orchestrate the new stack because we speak both languages fluently. We aren't just "using AI"; we are directing a Supernova of productivity using **Cursor** and **Claude Code**.

## 1. The Stack: The Loop and The Arc

Stop asking "Which one should I use?" That's a junior question. The Design Engineer uses both, effectively creating a **Supernova** of productivity.

*   **Cursor** is the **Loop** (The interactive, sub-100ms visual state). This is where we perfect the spring physics of a modal using **GSAP** and `useGSAP()`.
*   **Claude Code** is the **Arc** (The autonomous, multi-hour reasoning state). This is where we refactor the data layer while we sleep.

## 2. The Configuration Layer: Design Systems as Code

The difference between the "I code in Vim on a ThinkPad" purist and a Senior Design Engineer in 2026 isn't skill--it's their **Configuration Layer**. We don't guess; we encode our taste into the machine.

### Cursor: Context Gating via `.mdc`
We use the `.cursor/rules/` directory to force the AI to respect our design system tokens and motion standards.

```yaml
# .cursor/rules/design-system.mdc
---
description: Design System & GSAP Motion Standards
globs: src/components/**/*.tsx
---
# Animation Rules
- ALWAYS use `useGSAP()` hook for lifecycle management.
- Use `gsap.utils.selector(ref)` to prevent global scope leakage.
- Easing: Default to `expo.out`. Exits must be 30% faster than entrances.

# Token Enforcement
- Spacing: Use `gap-4` (1rem) steps. No arbitrary `mt-[13px]`.
- Colors: Use semantic tokens `text-monokai-pink` over hex codes.
```

### Claude Code: The Recursive Constitution
We treat the `.claude/` directory as our "System Architecture" documentation. We use custom slash commands to script complex agent behaviors.

*   **`CLAUDE.md`**: The project constitution. "We build fluid, optimistic UIs."
*   **`.claude/commands/ui-audit.md`**: A custom script that runs visual regression and accessibility audits via specialized subagents.

## 3. The "Hybrid Supercycle" in Action

Here is the exact workflow of a 10x Design Engineer in 2026.

### Phase 1: The Architect (Terminal)
We don't start by hacking CSS. We start by architecting the data flow that supports the UI.

**The Command:**
```bash
$ claude "I need a new 'Media Gallery' feature.
  1. Analyze `schema.prisma` and propose a `Media` model.
  2. Plan the tRPC routers for optimistic updates.
  3. Ensure the upload flow handles 50MB+ files.
  4. Generate a `specs/MEDIA_GALLERY.md`. Do NOT write code."
```

**The Output:**
Claude acts as the Senior Backend Engineer we don't have to hire. It creates a robust plan that ensures our beautiful UI won't crash the server.

### Phase 2: The Painter (Cursor Composer)
We switch to Cursor. This is our canvas. We open `specs/MEDIA_GALLERY.md`.

**The Action:**
Highlight the spec and hit `Cmd+I` (Composer).
> *"Scaffold the UI components based on this spec. Reference `.cursor/rules/design-system.mdc`. Use GSAP for the staggered entry of gallery items. I want the transition to feel 'glassy' and optimistic."*

**The Result:**
Because we gated the context with `.mdc` files, Cursor generates components that *already* look like our brand. We spend our time tweaking the `stagger` values and `ease` curves, not fixing import paths.

### Phase 3: The Auditor (Headless Terminal)
We finished the UI. It looks great. But is it solid? We don't manually check; we delegate.

**The Command:**
```bash
$ claude --print "Run the full test suite.
  If the new Media Gallery breaks any existing layout tests or accessibility checks:
  1. Read the error logs.
  2. Fix the component code.
  3. Verify the fix.
  Repeat until green."
```

While Claude fixes the regression in the background, we move on to designing the next interaction.

## 4. The Subagent Superpower

The Design Engineer knows they can't be an expert in everything. So we hire **Subagents**--specialized worker entities with isolated context windows.

*   **The A11y Subagent:** We pipe our component code into a specialized subagent that runs `axe-core` checks. It doesn't care about our "vibes"; it cares about WCAG 2.2 AA.
*   **The Documentation Subagent:** We use the **Context7 MCP** to fetch the latest GSAP 3.13+ documentation so our AI never hallucinates deprecated `TimelineLite` APIs.
